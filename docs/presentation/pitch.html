The quick experience

From git clone to a production image.

  1. Inspect. springdocker inspect reads pom.xml / build.gradle[.kts] and reports compatibility signals.
  2. Initialize. springdocker init writes a starter .springdocker.toml.
  3. Generate. springdocker dockerfile generate writes a Dockerfile and ensures .dockerignore.
  4. Verify. springdocker verify runs built-in checks (and optional plugin checks) with pass/fail output.
  5. Benchmark. springdocker benchmark generate/run/analyze provides reproducible evidence for tuning decisions.

What you get

A Dockerfile you'd write — if you had three days and a security team.


# springdocker · recipe: jvm-balanced · benchmark evidence included
# Java 21 · Spring Boot 3.3 · build-tool: maven · layered-jar: true

ARG BASE_BUILD=eclipse-temurin:21.0.5_11-jdk-jammy@sha256:7e2c…
ARG BASE_RUN=gcr.io/distroless/java21-debian12:nonroot@sha256:c9a1…

# ── build stage ─────────────────────────────────────────────────
FROM --platform=$BUILDPLATFORM ${BASE_BUILD} AS build
WORKDIR /app
COPY --link pom.xml mvnw ./
COPY --link .mvn ./.mvn
RUN --mount=type=cache,target=/root/.m2 ./mvnw -q dependency:go-offline
COPY --link src ./src
RUN --mount=type=cache,target=/root/.m2 ./mvnw -q -DskipTests package spring-boot:process-aot
RUN java -Djarmode=layertools -jar target/*.jar extract --destination /layers

# ── AppCDS training run ─────────────────────────────────────────
RUN java -XX:ArchiveClassesAtExit=/layers/app.jsa \
         -Dspring.context.exit=onRefresh \
         -cp /layers/application/BOOT-INF/classes:/layers/dependencies/BOOT-INF/lib/* \
         org.springframework.boot.loader.JarLauncher || true

# ── runtime stage ───────────────────────────────────────────────
FROM ${BASE_RUN}
LABEL org.opencontainers.image.source="$SOURCE" \
      org.opencontainers.image.revision="$REVISION" \
      org.opencontainers.image.created="$CREATED"
ENV TZ=UTC LANG=C.UTF-8
COPY --from=build --chown=nonroot:nonroot /layers/ /app/
USER nonroot
STOPSIGNAL SIGTERM
EXPOSE 8080
HEALTHCHECK --interval=10s --timeout=3s --start-period=20s --retries=3 \
  CMD ["/busybox/wget","-qO-","http://localhost:8080/actuator/health/readiness"]
ENTRYPOINT ["java","-XX:+UseZGC","-XX:+ZGenerational",\
  "-XX:MaxRAMPercentage=75","-XX:+ExitOnOutOfMemoryError",\
  "-XX:SharedArchiveFile=/app/app.jsa",\
  "org.springframework.boot.loader.JarLauncher"]
